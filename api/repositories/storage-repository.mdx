## Overview

The `StorageRepository` class manages file uploads and deletions in Firebase Cloud Storage. It handles image and audio file operations for stories.

## Usage Example

```kotlin
class AddStoryViewModel @Inject constructor(
    private val storageRepository: StorageRepository,
    private val storyRepository: StoryRepository
) : ViewModel() {

    fun uploadStoryWithImage(imageUri: Uri, storyData: Story) {
        viewModelScope.launch {
            // Generate temporary ID for storage path
            val tempId = UUID.randomUUID().toString()
            
            // Upload image
            val imageUrl = storageRepository.uploadImage(imageUri, tempId)
            
            if (imageUrl != null) {
                // Create story with image URL
                val story = storyData.copy(imageUrl = imageUrl)
                val storyId = storyRepository.addStory(story)
                // Handle success
            } else {
                // Handle upload error
            }
        }
    }
    
    fun deleteStoryFiles(story: Story) {
        viewModelScope.launch {
            story.imageUrl?.let { url ->
                storageRepository.deleteFile(url)
            }
            story.audioUrl?.let { url ->
                storageRepository.deleteFile(url)
            }
        }
    }
}
```

## File Organization

Files are organized in Firebase Storage as follows:

```
stories/
  {storyId}/
    cover.jpg    (story cover image)
    audio.mp3    (story narration audio)
```

## Error Handling

All methods return null or false on error rather than throwing exceptions. Common error scenarios:

- Network connectivity issues
- Insufficient permissions
- Invalid file URIs
- Storage quota exceeded
