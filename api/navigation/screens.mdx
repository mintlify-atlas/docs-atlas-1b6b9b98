## Overview

The `Screen` sealed class defines all navigation routes in the Casa de Historias app. Each screen has a constant route string used for navigation, with some screens supporting parameterized routes.

## Parameterized Screens

These screens accept route parameters:

### StoryDetail

```kotlin
object StoryDetail : Screen("story_detail/{storyId}") {
    fun createRoute(storyId: String) = "story_detail/$storyId"
}
```

Story detail view screen.

**Route Pattern:** `story_detail/{storyId}`

**Parameter:** `storyId` (String) - The unique identifier of the story to display

**Helper Method:**
- `createRoute(storyId: String): String` - Creates a route with the story ID filled in

**Source:** `~/workspace/source/app/src/main/java/com/example/casadehistorias/ui/navigation/NavGraph.kt:40`

**Usage Example:**
```kotlin
navController.navigate(Screen.StoryDetail.createRoute("story123"))
// Results in: "story_detail/story123"
```

## Navigation Examples

### Simple Navigation

```kotlin
// Navigate to a simple screen
navController.navigate(Screen.Login.route)
navController.navigate(Screen.Favorites.route)
navController.navigate(Screen.Search.route)
```

### Parameterized Navigation

```kotlin
// Navigate to story detail
val storyId = "abc123"
navController.navigate(Screen.StoryDetail.createRoute(storyId))

// Navigate to edit story
navController.navigate(Screen.EditStory.createRoute(storyId))
```

### Navigation with Pop Behavior

```kotlin
// Replace the entire back stack (e.g., after login)
navController.navigate(Screen.StoryList.route) {
    popUpTo(0) { inclusive = true }
}

// Pop up to a specific screen
navController.navigate(Screen.Login.route) {
    popUpTo(Screen.Welcome.route) { inclusive = true }
}
```

### Back Navigation

```kotlin
// Simple back navigation
navController.popBackStack()

// Navigate back to specific screen
navController.popBackStack(Screen.StoryList.route, inclusive = false)
```
