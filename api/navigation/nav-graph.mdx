## Overview

The `NavGraph` composable function defines the complete navigation structure for Casa de Historias. It sets up all navigation routes, their destinations, and navigation arguments.

## Declaration

```kotlin
@Composable
fun NavGraph(
    navController: NavHostController,
    authViewModel: AuthViewModel,
    startDestination: String
)
```

### Story Detail (Parameterized)

```kotlin
composable(
    route = Screen.StoryDetail.route,
    arguments = listOf(
        navArgument("storyId") { type = NavType.StringType }
    )
) { backStackEntry ->
    val storyId = backStackEntry.arguments?.getString("storyId") ?: ""
    val stories by storyViewModel.stories.collectAsState()
    val story = stories.find { it.id == storyId }

    LaunchedEffect(storyId) {
        if (story == null) {
            storyViewModel.loadStoryById(storyId)
        }
    }

    val selectedStory by storyViewModel.selectedStory.collectAsState()
    val displayStory = story ?: selectedStory

    if (displayStory != null) {
        StoryDetailScreen(
            story = displayStory,
            favoritesViewModel = favoritesViewModel,
            onNavigateBack = { navController.popBackStack() },
            onEditClick = {
                navController.navigate(Screen.EditStory.createRoute(storyId))
            }
        )
    } else {
        StoryDetailLoadingScreen(
            onNavigateBack = { navController.popBackStack() }
        )
    }
}
```

**Source:** `~/workspace/source/app/src/main/java/com/example/casadehistorias/ui/navigation/NavGraph.kt:187`

**Key Features:**
- Accepts `storyId` as a String navigation argument
- Tries to find story in local list first for instant display
- Falls back to loading from Firestore if not in local list
- Shows loading screen while fetching
- Passes shared `favoritesViewModel` for favorite functionality

## Usage Example

```kotlin
@Composable
fun MainActivity() {
    val navController = rememberNavController()
    val authViewModel: AuthViewModel = hiltViewModel()
    
    // Determine start destination based on auth state
    val startDestination = if (authViewModel.isLoggedIn) {
        Screen.StoryList.route
    } else {
        Screen.Welcome.route
    }
    
    NavGraph(
        navController = navController,
        authViewModel = authViewModel,
        startDestination = startDestination
    )
}
```

## Navigation Patterns

### Clear Back Stack Pattern

Used after successful login/signup to prevent back navigation to auth screens:

```kotlin
navController.navigate(Screen.StoryList.route) {
    popUpTo(0) { inclusive = true }
}
```

### Replace Current Screen Pattern

Used when transitioning from welcome to login:

```kotlin
navController.navigate(Screen.Login.route) {
    popUpTo(Screen.Welcome.route) { inclusive = true }
}
```

### Simple Forward Navigation

Used for most screen transitions:

```kotlin
navController.navigate(Screen.Search.route)
navController.navigate(Screen.StoryDetail.createRoute(storyId))
```

### Back Navigation

Used to return to previous screen:

```kotlin
navController.popBackStack()
```

## ViewModel Sharing

Certain ViewModels are created at the NavGraph level and shared across screens:

- `storyViewModel` - Shared between Discover, StoryList, StoryMap, and StoryDetail
- `favoritesViewModel` - Shared between Favorites and StoryDetail
- `authViewModel` - Passed as parameter, shared across all auth-related screens

This ensures consistent state across navigation and avoids redundant data loading.
