## Overview

The `UserProfile` data class represents a user's profile information stored in Firebase Firestore. It includes authentication details, community affiliation, storyteller status, and favorite stories.

## Declaration

```kotlin
data class UserProfile(
    @DocumentId
    val id: String = "",
    val email: String = "",
    val displayName: String = "",
    val photoUrl: String? = null,
    val community: String = "",
    val isStoryteller: Boolean = false,
    val favoriteStories: List<String> = emptyList(),
    val createdAt: Timestamp = Timestamp.now(),
    val lastLogin: Timestamp = Timestamp.now()
)
```

## Annotations

- `@DocumentId` - Firebase Firestore annotation that automatically maps the document ID to the `id` field

## Usage Example

```kotlin
import com.google.firebase.Timestamp

val userProfile = UserProfile(
    id = "user123",
    email = "maria@example.com",
    displayName = "María González",
    photoUrl = "https://storage.googleapis.com/profiles/user123.jpg",
    community = "Tepoztlán",
    isStoryteller = true,
    favoriteStories = listOf("story1", "story2", "story3"),
    createdAt = Timestamp.now(),
    lastLogin = Timestamp.now()
)
```

## Firebase Integration

This model is designed to work seamlessly with Firebase Firestore:

```kotlin
// Saving to Firestore
firestore.collection("users")
    .document(userId)
    .set(userProfile)
    .await()

// Reading from Firestore
val document = firestore.collection("users")
    .document(userId)
    .get()
    .await()
val profile = document.toObject(UserProfile::class.java)
```
