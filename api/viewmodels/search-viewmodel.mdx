## Debouncing Implementation

The SearchViewModel implements a 300ms debounce to avoid excessive search requests:

```kotlin
private var searchJob: Job? = null

fun onQueryChange(newQuery: String) {
    _query.value = newQuery
    searchJob?.cancel()  // Cancel previous search
    
    if (newQuery.isBlank()) {
        _results.value = emptyList()
        _isSearching.value = false
        return
    }
    
    searchJob = viewModelScope.launch {
        _isSearching.value = true
        delay(300)  // Wait 300ms
        val searchResults = repository.searchStories(newQuery)
        _results.value = searchResults
        _isSearching.value = false
    }
}
```

## Usage Example

```kotlin
@Composable
fun SearchScreen(
    viewModel: SearchViewModel = hiltViewModel()
) {
    val query by viewModel.query.collectAsState()
    val results by viewModel.results.collectAsState()
    val isSearching by viewModel.isSearching.collectAsState()

    Column {
        SearchBar(
            query = query,
            onQueryChange = { viewModel.onQueryChange(it) },
            onClear = { viewModel.clearSearch() }
        )
        
        when {
            isSearching -> LoadingIndicator()
            query.isBlank() -> SearchPrompt()
            results.isEmpty() -> NoResultsView()
            else -> LazyColumn {
                items(results) { story ->
                    StoryCard(story = story)
                }
            }
        }
    }
}
```

## Search Behavior

The search functionality matches stories across multiple fields:
- Spanish title (`titleEs`)
- Nahuatl title (`titleNahuatl`)
- Narrator name (`narratorName`)
- Community name (`community`)
- Tags

All searches are case-insensitive.

## Performance Optimization

Key optimizations implemented:

1. **Debouncing**: 300ms delay prevents excessive API calls while typing
2. **Job Cancellation**: Previous search is cancelled when query changes
3. **Early Return**: Blank queries immediately clear results without API call
4. **Coroutines**: Search executes in background without blocking UI
