## Overview

Casa de Historias uses Cloud Firestore as its primary cloud database for storing and syncing stories, user profiles, and application data in real-time. The `FirestoreRepository` class provides a comprehensive API for interacting with Firestore collections.

## FirestoreRepository

Location: `data/firebase/FirestoreRepository.kt:15`

The repository is implemented as a Singleton and uses dependency injection to access the FirebaseFirestore instance.

```kotlin
@Singleton
class FirestoreRepository @Inject constructor(
    private val firestore: FirebaseFirestore
)
```

## Core Operations

### Real-time Story Updates

Location: `data/firebase/FirestoreRepository.kt:19`

The repository provides a Flow-based API for real-time story updates:

```kotlin
fun getAllStoriesFlow(): Flow<List<StoryFirestore>> = callbackFlow {
    val listener = firestore.collection("stories")
        .whereEqualTo("published", true)
        .orderBy("createdAt", Query.Direction.DESCENDING)
        .addSnapshotListener { snapshot, error ->
            if (error != null) {
                close(error)
                return@addSnapshotListener
            }

            val stories = snapshot?.documents?.mapNotNull { doc ->
                doc.toObject(StoryFirestore::class.java)
            } ?: emptyList()

            trySend(stories)
        }

    awaitClose { listener.remove() }
}
```

<Tip>
Using `callbackFlow` allows seamless integration with Kotlin Coroutines and Compose, enabling reactive UI updates.
</Tip>

### Batch Queries

Location: `data/firebase/FirestoreRepository.kt:97`

Firestore has a limitation of 30 items per `whereIn` query. The repository handles this automatically:

```kotlin
suspend fun getStoriesByIds(ids: List<String>): List<StoryFirestore> {
    if (ids.isEmpty()) return emptyList()
    return try {
        // Firestore 'in' queries support max 30 items
        val results = mutableListOf<StoryFirestore>()
        ids.chunked(30).forEach { chunk ->
            val snapshot = firestore.collection("stories")
                .whereIn("__name__", chunk.map { firestore.collection("stories").document(it) })
                .get()
                .await()
            results.addAll(snapshot.documents.mapNotNull { doc ->
                doc.toObject(StoryFirestore::class.java)
            })
        }
        results
    } catch (e: Exception) {
        // Fallback: fetch one by one
        ids.mapNotNull { getStoryById(it) }
    }
}
```

<Warning>
Firestore limits `whereIn` queries to 30 items. The implementation chunks large ID lists and falls back to individual fetches if batch queries fail.
</Warning>

### Write Operations

#### Add Story

```kotlin
suspend fun addStory(story: StoryFirestore): String? {
    return try {
        val documentRef = firestore.collection("stories").document()
        val storyWithId = story.copy(
            id = documentRef.id,
            createdAt = Timestamp.now(),
            updatedAt = Timestamp.now()
        )
        documentRef.set(storyWithId).await()
        documentRef.id
    } catch (e: Exception) {
        null
    }
}
```

### Search Implementation

Location: `data/firebase/FirestoreRepository.kt:154`

Due to Firestore's limited full-text search capabilities, the app fetches all published stories and filters them locally:

```kotlin
suspend fun searchStories(query: String): List<StoryFirestore> {
    return try {
        // Fetch all published stories and filter locally for more flexible search
        val snapshot = firestore.collection("stories")
            .whereEqualTo("published", true)
            .get()
            .await()

        val allStories = snapshot.documents.mapNotNull { doc ->
            doc.toObject(StoryFirestore::class.java)
        }

        val lowerQuery = query.lowercase()
        allStories.filter { story ->
            story.titleEs.lowercase().contains(lowerQuery) ||
            story.titleNahuatl.lowercase().contains(lowerQuery) ||
            story.narratorName.lowercase().contains(lowerQuery) ||
            story.community.lowercase().contains(lowerQuery) ||
            story.tags.any { it.lowercase().contains(lowerQuery) }
        }
    } catch (e: Exception) {
        emptyList()
    }
}
```

<Note>
For production apps with large datasets, consider using Algolia, ElasticSearch, or Cloud Functions with full-text search capabilities.
</Note>

### Favorites Management

The repository manages user favorites through Firestore array operations:

#### Add to Favorites

```kotlin
suspend fun addToFavorites(userId: String, storyId: String): Boolean {
    return try {
        firestore.collection("users").document(userId)
            .update("favoriteStories", FieldValue.arrayUnion(storyId))
            .await()
        true
    } catch (e: Exception) {
        false
    }
}
```

#### Remove from Favorites

Location: `data/firebase/FirestoreRepository.kt:190`

```kotlin
suspend fun removeFromFavorites(userId: String, storyId: String): Boolean {
    return try {
        firestore.collection("users").document(userId)
            .update("favoriteStories", FieldValue.arrayRemove(storyId))
            .await()
        true
    } catch (e: Exception) {
        false
    }
}
```

<Tip>
`FieldValue.arrayUnion()` and `FieldValue.arrayRemove()` are atomic operations that prevent race conditions when multiple devices update favorites simultaneously.
</Tip>

## Error Handling

All repository methods use try-catch blocks and return safe defaults:

- Query methods return empty lists on error
- Single-item queries return `null` on error
- Write operations return `false` on failure
