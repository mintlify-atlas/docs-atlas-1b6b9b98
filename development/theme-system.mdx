## Overview

Casa de Historias uses a custom dark theme called **"Noche Mística"** (Mystical Night), inspired by ancestral nights in the Sierra de Zongolica. The theme leverages Material Design 3 with a culturally-resonant color palette.

## Color Palette

The app uses a custom color palette defined in `Color.kt` with meaningful names that evoke the app's cultural context.

## Theme Configuration

The theme is configured using Material Design 3's `darkColorScheme` in `Theme.kt`:

<CodeGroup>
```kotlin Theme.kt
private val CasaDeHistoriasColorScheme = darkColorScheme(
    primary = AguaSagrada,
    onPrimary = NocheProfunda,
    primaryContainer = AguaSagradaOscura,
    onPrimaryContainer = LunaLlena,

    secondary = FloresCeremoniales,
    onSecondary = LunaLlena,
    secondaryContainer = FloresCeremonialesOscuras,
    onSecondaryContainer = LunaLlena,

    background = NocheProfunda,
    onBackground = LunaLlena,
    surface = FondoTarjeta,
    onSurface = LunaLlena,

    outline = TextoSecundario,
    outlineVariant = TextoSecundario.copy(alpha = 0.5f),

    surfaceVariant = SurfaceVariant,
    onSurfaceVariant = TextoSecundario,

    error = FloresCeremoniales,
    onError = LunaLlena
)
```
</CodeGroup>

### Theme Composable

The `CasaDeHistoriasTheme` composable applies the theme and configures the system UI:

<CodeGroup>
```kotlin Theme.kt
@Composable
fun CasaDeHistoriasTheme(
    content: @Composable () -> Unit
) {
    val colorScheme = CasaDeHistoriasColorScheme

    val view = LocalView.current
    if (!view.isInEditMode) {
        SideEffect {
            val window = (view.context as Activity).window
            window.statusBarColor = NocheProfunda.toArgb()
            WindowCompat.getInsetsController(window, view)
                .isAppearanceLightStatusBars = false
        }
    }

    MaterialTheme(
        colorScheme = colorScheme,
        typography = Typography,
        content = content
    )
}
```
</CodeGroup>

<Note>
The theme automatically sets the status bar color to `NocheProfunda` and configures it for dark appearance.
</Note>

## Using the Theme

### In Screens

Access theme colors and typography through `MaterialTheme`:

<CodeGroup>
```kotlin Example Screen
@Composable
fun MyScreen() {
    Scaffold(
        containerColor = NocheProfunda
    ) { padding ->
        Column(
            modifier = Modifier.padding(padding)
        ) {
            Text(
                text = "Title",
                style = MaterialTheme.typography.headlineLarge,
                color = LunaLlena
            )
            Text(
                text = "Subtitle",
                style = MaterialTheme.typography.bodyMedium,
                color = TextoSecundario
            )
        }
    }
}
```
</CodeGroup>

### Custom Components

Apply theme colors consistently:

<CodeGroup>
```kotlin Custom Button
Button(
    onClick = { /* ... */ },
    colors = ButtonDefaults.buttonColors(
        containerColor = AguaSagrada,
        contentColor = NocheProfunda
    ),
    shape = RoundedCornerShape(16.dp)
) {
    Text("Action", fontWeight = FontWeight.Bold)
}
```
</CodeGroup>

## Best Practices

<AccordionGroup>
  <Accordion title="Always use theme colors">
    Use the named theme colors (`AguaSagrada`, `LunaLlena`, etc.) instead of hardcoded hex values to maintain consistency and enable easy theme changes.
  </Accordion>

  <Accordion title="Use MaterialTheme for dynamic colors">
    Access `MaterialTheme.colorScheme` for semantic colors that automatically adapt:
    ```kotlin
    Text(
        text = "Error",
        color = MaterialTheme.colorScheme.error
    )
    ```
  </Accordion>

  <Accordion title="Respect typography scales">
    Use the predefined typography styles from `MaterialTheme.typography` for consistent text rendering across the app.
  </Accordion>

  <Accordion title="Consider accessibility">
    Ensure sufficient contrast between text and backgrounds. The theme's color choices provide good contrast ratios for readability.
  </Accordion>
</AccordionGroup>
