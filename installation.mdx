## System Requirements

### Java Development Kit (JDK)

Casa de Historias requires **JDK 17** for compilation:

```kotlin build.gradle.kts:33
compileOptions {
    sourceCompatibility = JavaVersion.VERSION_17
    targetCompatibility = JavaVersion.VERSION_17
}
kotlinOptions {
    jvmTarget = "17"
}
```

<Steps>
  <Step title="Download JDK 17">
    Download from [Oracle](https://www.oracle.com/java/technologies/javase/jdk17-archive-downloads.html) or use [OpenJDK](https://jdk.java.net/17/)
  </Step>
  
  <Step title="Install JDK">
    Follow the installer instructions for your operating system
  </Step>
  
  <Step title="Verify Installation">
    Run `java -version` in your terminal to confirm JDK 17 is installed
  </Step>
</Steps>

```bash
java -version
# Should output: java version "17.0.x" or "17.x.x"
```

### Android Studio

Download and install the latest stable version of Android Studio:

<Card title="Download Android Studio" icon="download" href="https://developer.android.com/studio">
  Get Android Studio Ladybug or newer from the official Android developer site
</Card>

<Note>
The project uses **Android Gradle Plugin 8.7.2** and **Kotlin 2.0.21**, which require a recent version of Android Studio.

## Configuring Firebase

Casa de Historias uses Firebase for backend services. You'll need to set up a Firebase project:

<Steps>
  <Step title="Create Firebase Project">
    Go to [Firebase Console](https://console.firebase.google.com/) and create a new project
  </Step>
  
  <Step title="Add Android App">
    Register your Android app with package name: `com.example.casadehistorias`
  </Step>
  
  <Step title="Download google-services.json">
    Download the `google-services.json` configuration file
  </Step>
  
  <Step title="Add to Project">
    Place `google-services.json` in the `app/` directory of your project
  </Step>
  
  <Step title="Enable Services">
    In Firebase Console, enable:
    - Authentication (Email/Password provider)
    - Cloud Firestore
    - Firebase Storage
    - Analytics (optional)
  </Step>
</Steps>

## Running on Device/Emulator

### Setting Up Android Emulator

<Steps>
  <Step title="Open AVD Manager">
    Tools → Device Manager (or AVD Manager in older versions)
  </Step>
  
  <Step title="Create Virtual Device">
    Click "Create Device" and select a phone with API 26 or higher
  </Step>
  
  <Step title="Download System Image">
    Download an Android system image (API 26+, recommend API 35 for best compatibility)
  </Step>
  
  <Step title="Launch Emulator">
    Select the created device and click the play button
  </Step>
</Steps>

<Warning>
Ensure your emulator has at least 2GB RAM and supports Google Play services if you need Firebase functionality.
</Warning>

### Running on Physical Device

<Steps>
  <Step title="Enable Developer Options">
    On your Android device, go to Settings → About Phone and tap Build Number 7 times
  </Step>
  
  <Step title="Enable USB Debugging">
    Settings → Developer Options → Enable USB Debugging
  </Step>
  
  <Step title="Connect Device">
    Connect your device via USB and authorize the computer
  </Step>
  
  <Step title="Select Device">
    In Android Studio, select your device from the device dropdown and click Run
  </Step>
</Steps>

### Minimum Device Requirements

- **Android Version**: 8.0 Oreo (API 26) or higher
- **Storage**: At least 100MB free space for the app
- **Permissions**: Internet access, location access for map features

## Troubleshooting

### Common Issues

<Accordion title="Gradle Sync Failed">
  **Solution**: 
  - Verify JDK 17 is configured: File → Project Structure → SDK Location
  - Check internet connection for dependency downloads
  - Invalidate caches: File → Invalidate Caches → Invalidate and Restart
</Accordion>

<Accordion title="Missing google-services.json">
  **Error**: `File google-services.json is missing`
  
  **Solution**:
  - Ensure you've downloaded `google-services.json` from Firebase Console
  - Place it in the `app/` directory (not `app/src/`)
  - Sync Gradle files after adding the file
</Accordion>

<Accordion title="Build Takes Too Long">
  **Solution**:
  - Increase Gradle memory in `gradle.properties`: `org.gradle.jvmargs=-Xmx4096m`
  - Enable Gradle daemon and parallel execution
  - Consider using a build cache
</Accordion>

<Accordion title="KSP Annotation Processing Errors">
  **Error**: Issues with Hilt or Room annotation processing
  
  **Solution**:
  - Clean and rebuild: Build → Clean Project, then Build → Rebuild Project
  - Ensure KSP version matches Kotlin version: `ksp = "2.0.21-1.0.27"`
  - Check that `@HiltAndroidApp` is present in `CasaDeHistoriasApp.kt`
</Accordion>

<Accordion title="Firebase Authentication Errors">
  **Solution**:
  - Verify `google-services.json` matches your Firebase project
  - Enable Email/Password authentication in Firebase Console
  - Check package name matches: `com.example.casadehistorias`
</Accordion>

### Build Configuration Issues

If you encounter compilation errors:

```bash
# Clean build
./gradlew clean

# Rebuild project
./gradlew build

# Check for dependency updates
./gradlew dependencyUpdates
```
